Como reportar Bugs, Board Kanban - Aula 7

Board Kanban

Serve para ajudar nos processos de desenvolvimentos de funcionalidades, tanto para TI, quanto para qualquer área.

essa ferramenta faz uma gestao visual do que esta sendo desenvolvido pela squad e estima prazos de finalização do projeto

Backlog

É a coluna onde a area de produto em conjunto com o PO do time vai organizar qual é a ordem de prioridade as tarefas e historias que deverao ser testadas a seguir

Em Desenvolvimento

Indica que o desenvolvimento foi iniciado e embreve estará disponivel para os testes.

Testes

Aqui, o que foi desenvolvido é testato, seguindo os criterios de aceite definidos pelo PO, detalhando os cenarios que foram testados, incluindo fotos e arquivos.

Done

Aqui é onde finaliza o teste.

Observação

Não existe padrao de bord. Cada ferramenta usa uma quantidade de colunas com regras variadas para a transiçao de uma coluna para a outra. Tudo vai depender da necessidade do time que esta a trabalhar

Dicas

Tipos de Testes - Aula 8

Caixa preta: nao temos acesso ao codigo. Avalia o criterio de aceite, avalia se o que foi definido pela equipe de produtos esta de acordo no site. Avalia pela perspectiva do usuario

Caixa Branca: avalia o codigo do software. Avalia se este codigo está atendendo as especificações tecnicas, se esta de acordo com o que foi pedido no escopo do projeto, entre outros

Teste Funcional: é um tipo de teste de caixa preta e quem aplica são os QAs. Valida o software pela perspectiva do usuario, procurando bugs de forma usual, valida se todas as funcionalidades estao de acordo com o esperado. Pode ser manual ou automatizado

Sugestoes de ferramentas de Testes Automatizados:

Katalon, Appiun, TestComplete, Protractor, TestCafe, Selenium Webdrive, Cypress, entre outros.

Teste Unitario: é do tipo caixa branca e é aplicado pelos desenvolvedores. Nesse teste o dev testa a menor parte possivel do software, validando as entradas e saidas, usando frameworks.

Teste Componente\Instrumentado\UI: para o frontend com a ajuda de framewors valida a entrada e saida avaliando com componentes de entrada e saida

Teste Integrado: pode ser tanto de caixa branca quanto de caixa preta.Pode ser aplicada tanto pelos Devs quando pelos QAs. O teste pega as APIs distintas (todos os pedaços so software) e avalia como elas trabalham em conjunto.

Teste Regressivo ou de Regressão: ele pode ser cx preta ou branca. Aplicado pelos QAs. Retestar todo sofware, tanto unitários quanto Automatizados

Teste de Carga e Teste de Stresse: é um teste não funcional. Aplicado por equipes maiores, por ser um teste extremamente delicado. Ele valida quantos usuarios simultaneos podem acessar o site sem ele cair. Esse teste tem o intuido de evitar que o site saia do ar em momentos criticos como BlackFriday por exemplo.

Sugestao de ferramentas para testes de carga: Gatling, BlazeMeter e JMeter

Teste de Usabilidade: é aplicado normalmente pelo PO e pelos UX. Geralmente é aplicado entrevistando o usuário, visando melhorar a usabilidade do aplicativo.

Obs: as pessoas que aplicam o teste não são regra. Voce como QA pode fazer esses testes mencionados tambem.
