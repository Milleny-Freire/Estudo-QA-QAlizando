Como reportar Bugs, Board Kanban - Aula 7

Board Kanban

Serve para ajudar nos processos de desenvolvimentos de funcionalidades, tanto para TI, quanto para qualquer área.

essa ferramenta faz uma gestao visual do que esta sendo desenvolvido pela squad e estima prazos de finalização do projeto

Backlog

É a coluna onde a area de produto em conjunto com o PO do time vai organizar qual é a ordem de prioridade as tarefas e historias que deverao ser testadas a seguir

Em Desenvolvimento

Indica que o desenvolvimento foi iniciado e embreve estará disponivel para os testes.

Testes

Aqui, o que foi desenvolvido é testato, seguindo os criterios de aceite definidos pelo PO, detalhando os cenarios que foram testados, incluindo fotos e arquivos.

Done

Aqui é onde finaliza o teste.

Observação

Não existe padrao de bord. Cada ferramenta usa uma quantidade de colunas com regras variadas para a transiçao de uma coluna para a outra. Tudo vai depender da necessidade do time que esta a trabalhar

Dicas

Sempre abra os Bugs no gira, para ter historico e registro

Um card nunca volta a tarefa

Mantenha o Gira sempre atualizado
